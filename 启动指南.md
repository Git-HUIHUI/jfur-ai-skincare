# 晶肤AI美肤助手 — 本地启动指南（ngrok 公网共享版）

## 启动步骤

需要开 **三个** 终端窗口（或两个终端 + 一个 tab）：

### 终端 1：启动后端

```bash
cd D:\medical-aesthetics-ai-agent\backend
C:\Users\sxnss\.conda\envs\jfur-ai\python.exe -m uvicorn main:app --host 0.0.0.0 --port 8001
```

看到 `✅ 服务启动完成` 即可。

### 终端 2：启动前端

```bash
cd D:\medical-aesthetics-ai-agent\frontend
npm run dev
```

看到 `http://localhost:3000` 后，确认 `.env.local` 配置正确（见下方「环境变量」）。

### 终端 3：启动 ngrok 公网隧道

**方式 A（推荐）**：项目根目录运行脚本

```bash
cd D:\medical-aesthetics-ai-agent
C:\Users\sxnss\.conda\envs\jfur-ai\python.exe start_ngrok.py
```

**方式 B**：一行命令

```bash
C:\Users\sxnss\.conda\envs\jfur-ai\python.exe -c "from pyngrok import ngrok; print('公网地址:', ngrok.connect(3000, 'http').public_url)"
```

终端会输出类似 `https://xxx.ngrok-free.dev`，把这个链接发给别人即可访问。

## 公网访问原理（重要）

### 为什么不能浏览器直连 `localhost:8001`？

ngrok 只把 **前端 3000 端口** 暴露到公网。若前端让浏览器直接请求 `http://localhost:8001`：

- **本地访问**：`localhost` = 你的电脑 → 能连到你的 FastAPI ✅
- **公网访问**：`localhost` = **远程用户自己的电脑** → 上面没有你的后端 ❌

因此会出现错误：`无法连接到后端服务: http://localhost:8001`

### 正确请求链路

```
远程浏览器 → ngrok → 本机 Next.js (3000) → /api/chat 代理 → 本机 FastAPI (8001)
```

要点：

1. **后端和前端都要在本机运行**
2. **只分享 ngrok 的前端链接**，无需给后端单独开隧道
3. 浏览器只访问同源的 `/api/chat`，由 Next.js **在服务端**转发到 `localhost:8001`

### 相关代码

| 文件 | 作用 |
|------|------|
| `frontend/app/page.tsx` | 浏览器请求 `/api/chat`（不直连后端） |
| `frontend/app/api/chat/route.ts` | 服务端代理，转发到 `BACKEND_URL` |
| `frontend/.env.local` | 配置 `BACKEND_URL=http://localhost:8001`（仅服务端读取） |

## 环境变量

复制并编辑前端配置：

```bash
cd frontend
cp .env.example .env.local
```

`frontend/.env.local` 推荐内容：

```env
# 仅 Next.js 服务端代理使用
BACKEND_URL=http://localhost:8001
```

**不要**设置以下内容（会导致公网访问失败）：

```env
# ❌ 错误：此变量会注入到浏览器，远程用户会连自己的 localhost
NEXT_PUBLIC_API_URL=http://localhost:8001
```

修改 `.env.local` 或前端代码后，需 **重启** `npm run dev` 才能生效。

## 关闭

直接关掉三个终端窗口即可。

如果窗口关不掉，在任务管理器里结束 `python.exe`、`node.exe`、`ngrok.exe` 进程。

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 公网报 `localhost:8001` 连接失败 | 浏览器直连了本机地址 | 确认前端走 `/api/chat` 代理；检查 `.env.local` 无 `NEXT_PUBLIC_API_URL`；重启 `npm run dev` |
| 本地正常、公网不行 | 同上 | 同上 |
| 8001 端口被占用 | 已有后端实例在跑 | 可直接复用；或任务管理器结束旧 `python.exe` 后重启 |
| 3000 端口被占用 | 已有前端实例在跑 | Next.js 会自动改用 3001；此时 ngrok 也要指向 **实际端口**（改 `start_ngrok.py` 中的 `3000`） |
| ngrok 报 endpoint already online | 已有 ngrok 隧道在跑 | 查看现有地址：`curl http://127.0.0.1:4040/api/tunnels`；或结束旧 `ngrok.exe` 后重试 |
| npm run dev 报错 | 依赖未安装 | 先执行 `npm install` |
| ngrok 连不上 | 网络或账号问题 | 等 10 秒重试；确认 pyngrok / ngrok 已配置 authtoken |

## 快速自检

启动完成后，在本机依次验证：

```bash
# 1. 后端健康检查
curl http://localhost:8001/api/health

# 2. 前端可访问
curl -o NUL -w "%{http_code}" http://localhost:3000

# 3. 查看 ngrok 公网地址（若 ngrok 已在跑）
curl http://127.0.0.1:4040/api/tunnels
```
