晶肤医美 AI 智能体 Demo 技术栈文档（国内版）

项目背景
为晶肤医美（朗姿股份旗下轻医美连锁品牌）开发一个AI智能体Demo，用于面试展示。用户通过网页聊天界面上传面部照片并用文字描述需求，AI分析皮肤表面特征、追问细节、从晶肤项目知识库中匹配推荐方案、生成术后效果模拟图，最后引导预约线下门店。

所有AI模型使用国内供应商，无需国外API，无需信用卡或海外支付。


一、前端

框架
- Next.js 14 (App Router)
- React 18+
- TypeScript

AI 交互
- Vercel AI SDK (ai 和 @ai-sdk/react 包)
- 使用 useChat hook 实现：
  - 流式对话（打字机效果）
  - 多模态输入（文本 + 图片上传）
  - 自动处理消息状态

样式
- Tailwind CSS
- shadcn/ui 组件库（Button, Card, Input, Avatar 等）

部署
- Vercel 平台（免费自动分配域名，如 xxx.vercel.app）

界面结构
1. 顶部栏：晶肤 logo（左侧）+ 标题“晶肤AI美肤助手”（居中）
2. 聊天区域：消息列表，自适应高度，自动滚动到底部
3. 底部输入区：
   - 文本输入框（固定底部）
   - 图片上传按钮（支持点击选择文件和拖拽上传）
   - 发送按钮
4. 消息气泡类型：
   - 用户消息：靠右，浅色背景
   - AI 普通消息：靠左，白色背景，打字机逐字输出
   - AI 皮肤分析结果：靠左，带项目符号列表的卡片样式
   - AI 项目推荐卡片：靠左，带项目名称、描述、价格区间的卡片
   - AI 效果模拟图：靠左，内嵌图片，下方小字声明
   - AI 预约引导：靠左，带操作按钮的消息


二、后端

框架
- Python 3.10+
- FastAPI
- uvicorn

API 端点
- POST /api/chat：接收消息列表和图片文件，返回流式响应
- 请求体格式：multipart/form-data，包含 messages (JSON) 和 image (file)
- 响应格式：Server-Sent Events (SSE) 流式文本

环境变量管理
- python-dotenv

部署
- Railway 或 Render（任选一个免费托管）
- 需设置环境变量 DASHSCOPE_API_KEY


三、智能体编排（LangGraph）

框架
- LangGraph
- LangChain（辅助用）

状态图定义
定义 5 个核心节点 + 1 个人工审核检查点：

节点1：analyze（皮肤分析）
- 输入：用户上传的面部图片 + 文字描述
- 处理：调用 Qwen-VL-Max 多模态模型
- 输出：皮肤表面特征描述（毛孔粗大区域、痘印位置与颜色、肤色均匀度等）
- 严格约束：仅描述表面特征，禁止下医学诊断

节点2：ask_followup（追问确认）
- 输入：节点1的分析结果
- 处理：生成针对性的追问（如痘印是近期还是旧印、色斑是否晒后加重等）
- 输出：追问文本，等待用户回复后继续

节点3：match_product（方案匹配）
- 输入：用户确认后的需求信息
- 处理：将需求向量化，在 ChromaDB 中检索最匹配的晶肤项目
- 输出：推荐项目名称、描述、价格区间、搭配建议

检查点：human_review（人工审核模拟）
- 在节点3和节点4之间插入
- 使用 LangGraph Checkpointing 实现暂停/恢复
- Demo 中设置为自动通过，但保留代码逻辑

节点4：generate_preview（效果生成）
- 输入：用户原始照片 + 推荐项目对应的改善提示词
- 处理：调用通义万相生成效果模拟图
- 输出：生成的效果图 URL 或 base64 图片

节点5：book_appointment（预约引导）
- 输入：最终推荐方案
- 处理：返回晶肤门店列表和预约入口
- Demo 阶段使用 Mock 数据，返回“预约功能开发中，请到晶肤官网预约”

流程图
用户输入 → analyze → ask_followup（循环直到用户确认）→ match_product → [人工审核检查点] → generate_preview → book_appointment → 返回前端


四、多模态理解（国内模型）

模型
- 名称：Qwen-VL-Max
- 供应商：阿里云百炼（DashScope）
- 能力：支持图片+文本输入，输出文本描述

调用方式
使用 OpenAI SDK 兼容模式调用：

from openai import OpenAI

client = OpenAI(
    api_key="your-dashscope-api-key",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)

response = client.chat.completions.create(
    model="qwen-vl-max",
    messages=[
        {
            "role": "system",
            "content": "系统提示词（见下方）"
        },
        {
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": "图片的base64 data URI或URL"}},
                {"type": "text", "text": "用户的文字描述"}
            ]
        }
    ],
    stream=True
)

系统提示词（System Prompt）
你是晶肤医美的AI美肤助手，名字叫"小肤"。

你的能力
- 根据用户上传的面部照片，描述皮肤的表面特征（毛孔粗细、痘印位置与颜色、肤色均匀度、光泽度等）
- 介绍晶肤医美的轻医美项目，解释治疗原理和适用人群
- 根据用户需求匹配合适的项目组合

你的禁区（绝对不能做）
- 不能做出任何医学诊断（如“你这是黄褐斑/痤疮”）
- 不能承诺治疗效果
- 不能说“你不需要看医生”
- 不能编造项目名称或价格

你的回复风格
- 温暖、像朋友一样聊天
- 把专业知识用大白话解释
- 每次提到效果时，加上“效果因人而异，具体以医生面诊为准”
- 在推荐项目后，引导用户预约线下医生面诊

重要合规要求
所有分析结果仅作为初步参考，不能替代专业医生的面诊和皮肤检测。


五、产品匹配（RAG）

向量数据库
- ChromaDB（本地运行，无需联网）
- 存储路径：./chroma_db

Embedding 模型
- BAAI/bge-small-zh-v1.5
- 从 HuggingFace 本地加载，无需 API
- 向量维度：512
- 加载代码：
from sentence_transformers import SentenceTransformer
model = SentenceTransformer("BAAI/bge-small-zh-v1.5")

检索流程
1. 用户确认后的需求文本 → 用 BGE 模型向量化
2. 在 ChromaDB 中做相似度搜索（top_k=2）
3. 将检索到的项目文档作为上下文，注入 Qwen-VL-Max 的 Prompt
4. 模型基于真实资料生成推荐文案


六、效果模拟（图像生成）

模型
- 名称：通义万相（Text-to-Image）
- 供应商：阿里云百炼（DashScope）
- API 文档：DashScope 文本生成图像接口

调用方式
import dashscope
from dashscope import ImageSynthesis

dashscope.api_key = "your-api-key"

response = ImageSynthesis.call(
    model="wanx-v1",
    prompt="医美效果模拟：同一人，保持原五官和面部轮廓，仅改善皮肤：淡化红色痘印、提亮肤色、缩小T区毛孔，自然妆感，高清写实",
    n=1,
    size="1024*1024"
)
返回结果中包含图片URL

提示词模板
医美效果模拟图：保持原人物五官和面部轮廓完全不变，仅对皮肤做以下改善：[具体改善项]，自然效果，不过度美化，高清写实摄影

效果图展示要求
- 图片嵌入 AI 消息气泡中
- 图片下方必须带文字标注："以上为模拟效果，实际效果因人而异，以医生面诊为准"
- 生成过程中显示“正在生成效果模拟图...”加载状态

十、重要约束

- 所有 API 调用必须异步，不可阻塞流式输出
- 效果图生成时前端显示加载状态
- 所有 AI 分析结果标注“不是医学诊断”
- 所有项目推荐标注“以医生面诊为准”
- 所有效果图标注“模拟效果，实际因人而异”
- 代码关键节点加注释，方便面试展示时解释


十一、演示话术（面试时用）

这是我为晶肤医美开发的AI智能体Demo。前端用Next.js和Vercel AI SDK实现流式多模态对话，后端基于LangGraph做智能体编排，包含5个核心节点和1个人工审核检查点。多模态理解用阿里云百炼的Qwen-VL-Max，产品匹配用ChromaDB加BGE的RAG方案，效果生成用通义万相。整个系统严格遵循医疗合规要求，所有AI分析都标注不是诊断，引导用户必须线下面诊。这是一个可以直接部署的链接，您可以在手机上打开试试。
